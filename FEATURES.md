# RadReport Local — Функціональні вимоги

## MVP Features (Phase 1)

### F1: Голосовий запис та транскрипція (Append Mode)
- [ ] Кнопка "Почати запис" / "Зупинити запис" в toolbar
- [ ] Візуалізація waveform під час запису (audio level meter)
- [ ] Індикатор часу запису
- [ ] Відправка аудіо на Whisper API для транскрипції
- [ ] Відображення транскрибованого тексту в панелі "Диктовка"
- [ ] Підтримка української мови для транскрипції
- [ ] **Append Mode**: кожна нова диктовка ДОДАЄТЬСЯ до існуючого тексту (не затирає попередній)
- [ ] Візуальний роздільник між диктовками (горизонтальна лінія + timestamp: "Диктовка #2 — 14:35")
- [ ] Лічильник диктовок ("Диктовка 1 з 3")
- [ ] Кнопка "Очистити все" — видалення всіх диктовок (з підтвердженням)
- [ ] При натисканні "Обробити" — AI отримує ВЕСЬ накопичений текст усіх диктовок

### F2: Система шаблонів
- [ ] Завантаження шаблонів з JSON файлів
- [ ] Відображення списку доступних шаблонів (dropdown)
- [ ] Вибір шаблону перед початком роботи
- [ ] Відображення обраного шаблону з pre-filled нормою в лівій панелі
- [ ] Секції з підсекціями (вкладені findings)
- [ ] Початкові шаблони:
  - МРТ головного мозку
  - КТ органів грудної клітки
  - КТ органів черевної порожнини та заочеревинного простору

### F3: AI-маппінг (core feature)
- [ ] Кнопка "Обробити" — відправка транскрипції + шаблону на LLM
- [ ] LLM повертає structured JSON з маппінгами
- [ ] Парсинг відповіді та розподіл по секціях
- [ ] Заповнення фінального звіту на основі маппінгу
- [ ] Відображення unmapped фрагментів окремо
- [ ] Індикатор обробки (loading state)
- [ ] Обробка помилок API

### F4: Traceability — двонаправлене прослідковування
- [ ] Hover на секцію звіту → підсвітка джерельного фрагменту диктовки
- [ ] Hover на фрагмент диктовки → підсвітка цільової секції звіту
- [ ] Кольорове кодування (сірий/синій/зелений/помаранчевий)
- [ ] Tooltip з деталями маппінгу
- [ ] Плавна анімація підсвітки (transition)

### F5: Фінальний звіт та фіналізація

- [ ] Візуальне розрізнення: норма vs змінено AI
- [ ] Можливість ручного редагування кожної секції (inline edit)
- [ ] **Кнопка "Фіналізувати звіт"** (яскрава, помітна, в toolbar):
  - Збирає весь звіт з усіх секцій у форматований plain text

  - Показує toast-повідомлення "Звіт скопійовано в буфер обміну"
  - Після фіналізації — пропонує "Новий звіт" (очистити робочий простір)

- [ ] Кнопка "Новий звіт" — скидає стан (очищує диктовку, маппінг, повертає шаблон до дефолту)
- [ ] Діалог підтвердження перед очищенням ("Ви впевнені? Поточний звіт буде втрачено")

## User Stories

### US1: Базовий воркфлоу
> Як радіолог, я хочу обрати шаблон "МРТ головного мозку", надиктувати свої знахідки голосом, і отримати структурований звіт з розподіленими по секціях знахідками за кілька секунд.

### US2: Traceability
> Як радіолог, я хочу навести курсор на секцію "Шлуночкова система" у звіті і побачити, який саме фрагмент моєї диктовки був використаний для цієї секції, щоб переконатися в точності AI.

### US3: Unmapped text
> Як радіолог, я хочу бачити фрагменти моєї диктовки, які AI не зміг розмістити в жодну секцію шаблону, щоб нічого не було втрачено.

### US4: Ручне редагування
> Як радіолог, я хочу відредагувати будь-яку секцію фінального звіту вручну, якщо AI неточно інтерпретував мою диктовку.

### US5: Шаблони
> Як радіолог, я хочу створити власний шаблон для КТ ангіографії аорти з потрібними мені секціями і дефолтним текстом норми.

### US6: Множинні диктовки
> Як радіолог, я хочу надиктувати знахідки по паренхімі мозку, потім перейти до іншої серії зображень, надиктувати знахідки по шлуночках, і щоб обидва фрагменти диктовки зберіглися та були оброблені разом.

### US7: Фіналізація звіту
> Як радіолог, після того як я переглянув та відредагував звіт, я хочу натиснути одну кнопку "Фіналізувати", щоб готовий звіт скопіювався в буфер обміну і я міг вставити його у Word або госпітальну систему.

### US8: Новий пацієнт
> Як радіолог, після фіналізації звіту я хочу швидко почати роботу з новим пацієнтом — очистити робочий простір і повернути шаблон до стану норми.
