# RadReport Local — Архітектура додатку

## Огляд

**RadReport Local** — локальний Electron desktop-додаток для радіологів, який трансформує голосову диктовку в структуровані радіологічні звіти за допомогою AI.

Натхнення: [deepcOS® AIR®](https://www.deepc.ai/products/applications/air), [RADPAIR](https://radpair.com/), [VoxRad](https://github.com/drankush/VoxRad)

---

## Ключові можливості

### 1. Голосова диктовка з вбудованим мікрофоном
- Запис голосу безпосередньо в додатку через Web Audio API
- Транскрипція через OpenAI Whisper API (або OpenRouter-сумісний endpoint)
- Відображення waveform під час запису
- Підтримка української мови

### 2. AI-маппінг диктовки на шаблон
- Транскрибований текст відправляється на LLM (через OpenRouter API)
- LLM аналізує текст і розподіляє фрагменти по секціях шаблону
- Кожна секція шаблону має pre-filled текст "норми"
- AI або підтверджує норму, або замінює текст на основі диктовки

### 3. Traceability — двонаправлене прослідковування
- Hover на секцію звіту → підсвічується джерельний фрагмент диктовки
- Hover на фрагмент диктовки → підсвічується цільова секція звіту
- Кольорове кодування: норма / змінено AI / невикористаний текст

### 4. Система шаблонів
- JSON файли з структурованими секціями
- Вбудований редактор шаблонів
- Pre-filled "нормальні" знахідки для кожної секції
- Підтримка різних модальностей (МРТ, КТ, УЗД, Рентген)

### 5. Редактор шаблонів
- Створення нових шаблонів через UI
- Додавання/видалення секцій та підсекцій
- Редагування дефолтного тексту норми
- Імпорт/експорт шаблонів

## Data Flow

```
1. [Мікрофон] → Web Audio API → AudioBlob
                                    │
2. AudioBlob → Whisper API ──────→ Raw Transcript (plain text)
                                    │
3. Transcript ДОДАЄТЬСЯ до накопиченого тексту (Append Mode):
   dictationHistory: [
     { id: 1, text: "...", timestamp: "14:30" },
     { id: 2, text: "...", timestamp: "14:35" },  ← нова диктовка
   ]
   fullTranscript = dictationHistory.map(d => d.text).join("\n")
                                    │
4. fullTranscript + Template JSON → LLM (OpenRouter)
                                    │
5. LLM повертає StructuredMapping JSON:
   {
     mappings: [
       { source: {text, start, end}, target: {section, action, newText} }
     ],
     unmapped: [
       { text, start, end, reason }
     ]
   }
                                    │
6. StructuredMapping → UI Render:
   - Template Panel: секції з підсвіткою змін
   - Dictation Panel: фрагменти з кольоровим кодуванням
   - Report Panel: фінальний структурований звіт
   - Unmapped Panel: невикористані фрагменти
                                    │
7. [Фіналізувати] → збір тексту з усіх секцій → форматування → clipboard
                                    │
8. [Новий звіт] → reset стану → шаблон повертається до дефолтної норми
```

### Append Mode — множинні голосові введення

Радіолог може робити **декілька окремих записів** під час роботи з одним пацієнтом.
Кожен новий запис додається до вже існуючого тексту, а не затирає його.

```
Диктовка #1 (14:30): "Паренхіма мозку без вогнищевих змін..."
                           +
Диктовка #2 (14:35): "На серії DWI визначається зона рестрикції..."
                           +
Диктовка #3 (14:37): "Висновок: ознаки гострого ішемічного інсульту..."
                           =
Повний текст для AI-маппінгу (конкатенація всіх диктовок)
```

При натисканні "Обробити" — AI отримує **весь накопичений текст** усіх диктовок.

### Кольорове кодування

| Колір | Значення | Де |
|-------|----------|----|
| Сірий фон | Секція не змінена (норма підтверджена) | Template Panel, Report Panel |
| Синій фон | Секція змінена AI на основі диктовки | Report Panel |
| Зелений фон | Фрагмент диктовки успішно mapped | Dictation Panel |
| Помаранчевий фон | Фрагмент НЕ використаний | Dictation Panel, Unmapped Panel |
| Фіолетова рамка | Hover-highlight (traceability) | Всі панелі |

### Hover Traceability
- Наводиш курсор на секцію звіту → фіолетова підсвітка на відповідному фрагменті диктовки
- Наводиш курсор на фрагмент диктовки → фіолетова підсвітка на цільовій секції звіту
- Tooltip показує деталі: джерело → ціль

## Система шаблонів — Дизайн-рішення

### Прийняті рішення

| Питання | Рішення | Обґрунтування |
|---------|---------|---------------|
| Глибина вкладеності | **2 рівні** (section → subsection) | Достатньо для всіх модальностей. Підсекції покривають анатомічні структури (напр. передміхурова залоза → транзиторна зона / периферична зона, лімфатичні вузли → регіональні / екстрарегіональні) |
| Формат зберігання | **Окремі JSON файли** в папці `templates/` | Простота, git-версіонування, легко редагувати. Для MVP більш ніж достатньо |
| Початковий контент | **Пізніше** — на основі реальних звітів лікаря | Для MVP — placeholder шаблони зі спрощеним текстом. Реальні шаблони формуватимуться з наданих лікарем прикладів звітів |
| Формат фіналізації | **Dual clipboard** — plain text + HTML | Word підхопить HTML (bold заголовки), прості системи — plain text |

## Ключові типи даних

### DictationEntry (множинні диктовки)

```typescript
interface DictationEntry {
  id: number;
  text: string;                // транскрибований текст цієї диктовки
  timestamp: string;           // "14:35"
  createdAt: Date;
  audioLengthSeconds: number;  // тривалість запису
}

interface DictationState {
  entries: DictationEntry[];   // масив усіх диктовок
  isRecording: boolean;
  isTranscribing: boolean;

  // Повний текст для AI (конкатенація всіх entries)
  getFullTranscript(): string;

  // Дії
  addEntry(text: string, audioLength: number): void;
  clearAll(): void;
}
```

### Workflow State (lifecycle пацієнта)

```typescript
type WorkflowStage =
  | 'template_selected'    // шаблон обрано, готовий до диктовки
  | 'dictating'            // йде запис або є хоча б одна диктовка
  | 'processing'           // AI обробляє текст
  | 'reviewing'            // звіт сформований, можна редагувати
  | 'finalized';           // звіт фіналізовано та скопійовано

interface WorkflowState {
  stage: WorkflowStage;
  canRecord: boolean;        // true якщо template_selected | dictating | reviewing
  canProcess: boolean;       // true якщо є хоча б одна диктовка
  canFinalize: boolean;      // true якщо reviewing (звіт сформований)
  canReset: boolean;         // true завжди (окрім processing)
}
```

## AI Prompt Strategy

### Системний промт для LLM (маппінг диктовки → шаблон)

LLM отримує:
1. **Шаблон** — JSON з усіма секціями та дефолтним текстом
2. **Транскрипцію** — raw text від Whisper
3. **Інструкцію** — як виконати маппінг

LLM повертає **structured JSON** з:
- Масивом маппінгів (source fragment → target section + action)
- Масивом unmapped фрагментів
- Фінальним текстом кожної секції

### Типи дій (actions):
- `confirmed_normal` — лікар підтвердив норму голосом, текст не змінюється
- `replaced` — лікар описав патологію, дефолтний текст замінюється
- `appended` — додаткова інформація додана до існуючого тексту
- `not_mentioned` — лікар не згадав цю секцію (залишається дефолт)

### Транскрипція — Локальний Whisper

**Рішення**: використовуємо `whisper.cpp` з GGML-моделлю Large v3 **локально на Mac**.

```
models/
  ggml-large-v3.bin    # ~3 GB, завантажується один раз, НЕ йде в git
```

**Чому не MacWhisper моделі?** MacWhisper зберігає моделі у форматі WhisperKit CoreML (`.mlmodelc`) —
це Swift/CoreML runtime, недоступний з Node.js. Для Electron потрібен GGML формат для whisper.cpp.

**Pipeline транскрипції:**
1. Web Audio API записує аудіо → зберігає як WAV (16kHz mono)
2. Electron main process запускає whisper.cpp CLI або nodejs-whisper
3. whisper.cpp транскрибує аудіо локально на Apple Silicon (Neural Engine / GPU)
4. Текст повертається в renderer через IPC

**Переваги:**
- Безкоштовно (без API-викликів)
- Офлайн (не потрібен інтернет для транскрипції)
- Приватно (аудіо не покидає Mac)
- Швидко на Apple Silicon

## Безпека

- API ключ OpenRouter зберігається в `electron-store` з шифруванням
- **Транскрипція (Whisper) — повністю локальна**, аудіо не покидає Mac
- Дані пацієнтів НЕ зберігаються (лише шаблони та налаштування)
- Аудіо не зберігається після транскрипції
- AI-маппінг (LLM) відбувається через OpenRouter — **увага: транскрибований текст передається в хмару**
- В майбутньому: можливість локальної LLM (Ollama) для повної приватності
